## ğŸ—ï¸ Architecture Principles

Based on Zhamak Dehghani's four foundational pillars:

1. **Domain-oriented decentralized data ownership** - Data owned by domain teams who understand it best
2. **Data as a product** - Treating data with product thinking (quality, SLAs, discoverability)
3. **Self-serve data infrastructure as a platform** - Enable domain autonomy without requiring deep infrastructure expertise
4. **Federated computational governance** - Automated policies, not manual processes
